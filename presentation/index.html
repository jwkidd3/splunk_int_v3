Module 1

Advanced Search Commands

Subsearches

Use the results of one search to filter another

index=training sourcetype=web_access
    [search index=training sourcetype=security_events
     event_type="login_failure"
    | stats count by user
    | return 100 user]
| stats count by user, status

Key Points:

  • Runs inner search first
  • Limited to 50,000 results
  • Use return to control output

Transaction Command

Group related events into transactions

index=training sourcetype=web_access user!="-"
| transaction user clientip maxpause=30m
| table user, clientip, duration, eventcount

Options

  • maxpause - Max time between events
  • maxspan - Max transaction duration
  • startswith/endswith - Define boundaries

Use Cases

  • User sessions
  • Application workflows
  • Multi-step processes

Multisearch Command

Run multiple searches in parallel

| multisearch
    [search index=training sourcetype=web_access
     | eval source="Web"]
    [search index=training sourcetype=application
     | eval source="App"]
| stats count by source

Benefit: Faster than separate searches + append

Module 2

Statistical Commands & Transformations

Stats Command

Most flexible aggregation command

index=training sourcetype=web_access
| stats count as requests,
        dc(clientip) as unique_ips,
        avg(response_time) as avg_rt,
        perc95(response_time) as p95_rt
        by method, status

Common Functions:

count, sum, avg, min, max, dc, values, list, perc##, stdev

Chart vs. Timechart

Chart

| chart count
  over url
  by status

Two-dimensional aggregation (pivot table)

Timechart

| timechart
  span=1h
  count
  by status

Time-series specific aggregation

Eval Command

Transform and calculate fields

index=training sourcetype=web_access
| eval performance_score = case(
    response_time < 100 AND status < 400, 100,
    response_time < 500 AND status < 400, 75,
    response_time < 1000 AND status < 400, 50,
    status >= 400, 0,
    1=1, 25
)
| stats avg(performance_score) by user

String Functions

len, lower, upper, substr, replace

Math Functions

round, ceil, floor, abs, sqrt

Fields & Field Extractions

Field Extraction Methods

Automatic

  • Key-value pairs
  • JSON
  • XML
  • CSV

Manual

  • Rex command (inline)
  • Field Extractor UI
  • props.conf (persistent)
  • Calculated fields

Rex Command

Extract fields using regular expressions

index=training sourcetype=application
| rex field=message "order #(?<order_id>\d+)"
| rex field=message "(?<action>created|shipped|delivered)"
| stats count by order_id, action

Regex Groups:

(?<field_name>pattern) - Named capture group

Calculated Fields

Create persistent derived fields

Settings → Fields → Calculated Fields

Example: status_category

case(
    status < 300, "Success",
    status < 400, "Redirect",
    status < 500, "Client Error",
    status >= 500, "Server Error"
)

Benefit: Available automatically in all searches

Module 4

Lookups & Data Enrichment

Lookup Types

CSV Lookups

  • Static files
  • Good for small datasets
  • Easy to manage
  • < 1GB recommended

KV Store

  • MongoDB-based
  • Large datasets
  • REST API access
  • > 100K rows

Using Lookups

index=training sourcetype=web_access user!="-"
| lookup users.csv username as user
  OUTPUT department, city, role
| stats count by department, role

Automatic Lookups

Configure lookups to run automatically

Settings → Lookups → Automatic lookups

No need for | lookup command!

Inputlookup & Outputlookup

Inputlookup

| inputlookup users.csv
| table username,
        department

Read lookup table

Outputlookup

index=training
| stats count by user
| outputlookup
  user_stats.csv

Write to lookup table

Alerts & Scheduled Searches

Alert Types

Scheduled

Run on a schedule (cron)

Use for:

  • Batch analysis
  • Trend detection
  • SLA monitoring

Real-time

Continuous monitoring

Use for:

  • Security threats
  • Critical errors

Alert Actions

  • Run a script - Execute custom automation

Prevent alert fatigue

Throttling Options

Module 8

Data Models & Pivot

What are Data Models?

Hierarchical structures representing your data

  • Objects: Datasets (tables)
  • Fields: Attributes (columns)
  • Relationships: Parent-child links
  • Constraints: Filters defining scope

Benefits: Standardization, Performance, User-friendly access

Pivot Interface

Point-and-click data analysis (no SPL required)

Configure

  • Split Rows
  • Split Columns
  • Values (metrics)
  • Filters

Output

  • Tables
  • Charts
  • Statistics
  • Trends

Data Model Acceleration

What is Acceleration?

Pre-computed summary tables for instant results

  • Dramatically faster searches
  • Enable tstats command
  • Trade storage for speed
| tstats count from datamodel=Web_Traffic
  by Web_Access.method, Web_Access.status
| rename Web_Access.* as *

Module 9

Advanced Dashboard Techniques

Simple XML

Dashboard source code for advanced customization

<dashboard>
  <label>Dashboard Title</label>
  <row>
    <panel>
      <title>Panel Title</title>
      <search>
        <query>index=training | stats count</query>
      </search>
      <option name="charting.chart">line</option>
    </panel>
  </row>
</dashboard>

Base Searches

Share search results across panels

<search id="base_search">
  <query>index=training sourcetype=web_access</query>
</search>

<panel>
  <search base="base_search">
    <query>| stats count by status</query>
  </search>
</panel>

<panel>
  <search base="base_search">
    <query>| stats count by method</query>
  </search>
</panel>

Benefit: Run base search once, use multiple times

Form Dashboards

Features

  • Submit button control
  • Prevent automatic search execution
  • Better for complex inputs
  • Improved user control
<form version="1.1">
  <fieldset submitButton="true" autoRun="false">
    <!-- Inputs here -->
  </fieldset>
  <!-- Panels here -->
</form>
